Detailed Description

Utility class to send action commands.

PvActionCommand adds the ability to fire action commands, as defined in the GigE Vision™ standard.

To use a PvActionCommand, you instantiate an object from the class, configure the command to send, send the command, and then optionally retrieve statistics about the outcome of the command. It is possible to use the same action command object to send more than one command. To send the same command, call Send as many times as required. To send different commands, change the configuration of the command between Send calls.

The eBUS SDK does not configure action commands on the device directly. Instead, the SDK assumes that the user is manually configuring action commands with the GenICam® browser in the GEVPlayer sample application, or is configuring action commands programmatically (assuming that the device’s GenICam interface allows the user to configure the device directly).

PvResult PvActionCommand::Resend ( PvUInt32  aTimeout,
PvUInt32  aDeviceCount = 0,
bool  aRequestAcknowledges = true 
)

Resends the action command using the same request ID.

An application receiving an action command with the same request ID will acknowledge it but not execute the action if it was previously received.

Resends are not managed automatically as all application may have different requirements when it comes to resending actions. They need to be explicitly managed and performed if acknowledges are missing.

See Send

Parameters:
[in] aTimeout Maximum time to wait for answers in ms.
[in] aDeviceCount If different than zero, Send can return as soon as aDeviceCount acknowledges have been received without having to wait for aTimeout.
[in] aRequestAcknowledges Asks the devices for acknowledges of the action command.
Returns:
Includes:
  • PvResult::Code::OK
  • PvResult::Code::NOT_INITIALIZED If the command was never sent before calling Resend
  • PvResult::Code::STATE_ERROR If the content of the command has changed since the last Send operation. A resend cannot be performed if the command is not exactly the same.
  • PvResult::Code::TIMEOUT If the provided timeout expired before the reception of all the acknowledges

PvResult PvActionCommand::Send ( PvUInt32  aTimeout,
PvUInt32  aDeviceCount = 0,
bool  aRequestAcknowledges = true 
)

Broadcasts the action command on all enabled network interfaces.

If more than one network interface is enabled, broadcasts are sent on each network interface (one after the other, in the order they were added to the action command object) and we then wait for up to aTimeout ms for acknowledges on all these network interfaces.

When send is used, the request ID of the action command is automatically incremented. Use Resend to send the same action without incrementing the request ID.

The action command itself does not know how many devices are reached with this broadcast: all devices receiving the action command and configured to react to it (combination of device key, group key and group mask) are expected to perform the task associated with it.

See Resend

Parameters:
[in] aTimeout Maximum time to wait for answers in ms.
[in] aDeviceCount If different than zero, Send can return as soon as aDeviceCount acknowledges have been received without having to wait for aTimeout.
[in] aRequestAcknowledges Asks the devices for acknowledges of the action command.
Returns:
Includes:
  • PvResult::Code::OK
  • PvResult::Code::TIMEOUT If the provided timeout expired before the reception of all the acknowledges
void PvActionCommand::SetDeviceKey ( PvUInt32  aDeviceKey  ) 

Sets the current action command device key.

The device key is a 32-bit value used to check the validity of action commands. Devices only process action commands if the action command’s device key matches the device’s device key – think of it as a secret password provided by action commands that must match the password configured on the device. The device key on the device is typically configured using the GenICam® interface of the device. The device key is defined at the device level.

See GetDeviceKey

Parameters:
[in] aDeviceKey Device Key.
void PvActionCommand::SetGroupKey ( PvUInt32  aGroupKey  ) 

Sets the current action command group key.

The group key defines a group of devices on which actions have to be executed. If a device supports action 1 with a group key of 0x24 and action 2 with a group key of 0x42, when an action command with group key 0x24 is sent, only action 1 is executed. The group key for device actions is typically configured using the GenICam® interface of a device. The group key is defined at the action level: different actions on a device can have a different group keys.

See GetGroupKey

Parameters:
[in] aGroupKey Group Key.
void PvActionCommand::SetGroupMask ( PvUInt32  aGroupMask  ) 

Sets the current action command group mask.

A group mask can be configured for an action command. This mask is applied by the device against the group mask of the action command. If the resulting value is non-zero, the action is executed by the device. The group mask for device actions is typically configured using the GenICam® interface of the device. The group mask is defined at the action level: different actions on a device can have different group masks.

See GetGroupMask

void PvActionCommand::SetScheduledTime ( PvUInt64  aScheduledTime  ) 

Sets the scheduled time for the action command.

Action commands can either be performed on reception (no scheduled time enabled or provided) or they can be scheduled in the future. If a timestamp (device time domain) is provided, the device queues the action command and performs the associated task as close to the scheduled action command timestamp as possible. If the device receives a timestamp set in the past relative to its time domain, it is expected to execute the command on reception. The queue the device uses to hold future actions may be limited. If a new command is received and the device is not able to add it to a full queue, the sender is notified (if an acknowledgement of the action command was requested). The scheduled time is only used when the SetScheduledTimeEnable is True.

See GetScheduledTime, SetScheduledTimeEnable

Parameters:
[in] aScheduledTime If set to zero, a regular (non-scheduled) action command is issued. If different than zero, a GVCP scheduled action command with a 64-bit action time of aScheduledTime is issued.
void PvActionCommand::SetScheduledTimeEnable ( bool  aEnabled  ) 

Sets the scheduled time for the action command object.

Action commands can either be performed on reception (no scheduled time enabled and provided) or they can be scheduled for execution at a specific time in the future. This function allows the user to control when the scheduled action command is enabled.

See GetScheduledTimeEnable

Parameters:
[in] aEnabled true to use scheduled action command; false to use action command and disregard the scheduled time.
